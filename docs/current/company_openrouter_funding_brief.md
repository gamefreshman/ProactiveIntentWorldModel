# PIWM 实验进度与 OpenRouter 模型对比预算说明

更新时间：2026-05-01 CST

## 1. 当前进度

PIWM 目前已经完成从数据生产到模型训练评估的第一轮闭环：

```text
合成店内导购视频
-> 抽取关键帧
-> 人工/规则 QA
-> 结构化训练数据
-> Qwen2.5-VL LoRA SFT
-> 模型评估
```

这说明项目已经从“方案设计”推进到“有数据、有训练、有结果”的阶段。

## 2. 已有数据资产

| 类型 | 当前规模 | 用途 |
|---|---:|---|
| 合成导购视频训练集 | 260 条 parent videos | 主训练数据 |
| SFT 训练样本 | 1321 条 examples / 3963 张图像引用 | 训练 AI 导购模型 |
| 人工 QA 行为评估集 | 40 条审阅 / 36 条通过 | 验证模型是否看懂顾客行为 |
| 未来反应视频数据 | 44 条通过 QA | 验证模型能否理解动作后的顾客反应 |
| Future Verification 样本 | 84 对 | 验证“动作 + 未来反应视频”是否匹配 |

另外，扩展到 **500 / 1000 条 parent videos** 的 prompt 队列已经准备好，不需要重新设计数据管线。

## 3. 第一轮模型结果

我们用当前数据训练了一个 Qwen2.5-VL-7B LoRA 版本，结果显示训练是有效的。

| 能力 | 原始 Qwen2.5-VL | PIWM-SFT 后 | 说明 |
|---|---:|---:|---|
| 输出能否被系统稳定读取 | 23.5% - 55.2% | 100% | 从不稳定自然语言输出变成稳定结构化输出 |
| 识别顾客购买阶段 | 不稳定 | 88.9% | 可以初步判断顾客处于什么购买阶段 |
| 生成候选导购动作 | 不稳定 | 75.0% | 已能给出可用的导购动作候选 |
| 推演导购动作后果 | 约 26% - 37% | 约 99% - 100% | 能判断某个导购动作后可能带来的风险/收益 |
| 未来反应验证 | zero-shot 不稳定 | 明显提升 | continuation 视频不只是展示材料，已经能进入监督 |

结论：第一轮实验已经证明，PIWM 数据能把通用 VLM 往“AI 导购决策模型”方向有效拉动。

## 4. 现在还不能直接上线的原因

当前模型可以做 **受控 Demo / POC**，但还不建议直接作为无人值守导购上线。

主要原因：

- QA-reviewed 评估集还需要扩大；
- 当前状态识别和候选动作预测仍有提升空间；
- 只验证了 Qwen2.5-VL 一个主基座，还不知道其他商用/开源 VLM 是否更强；
- 需要比较效果和调用成本，才能确定产品化模型路线。

## 5. 为什么需要 OpenRouter 预算

下一步不是重新做 pipeline，而是做 **同一套数据协议下的多模型对比**。

OpenRouter 可以帮助我们快速比较多个 VLM：

| 对比内容 | 目标 |
|---|---|
| 不同 VLM 的 zero-shot 表现 | 判断哪些模型天然更懂店内导购场景 |
| 不同 VLM 的结构化输出能力 | 判断哪些模型更容易接入业务系统 |
| 不同 VLM 的视觉理解能力 | 判断哪些模型真的能看懂顾客行为 |
| 不同 VLM 的动作后果推理能力 | 判断哪些模型适合做 proactive sales assistant |
| 效果 / 成本对比 | 选择后续产品化和论文实验最值得押注的模型 |

本轮不优先测试 Claude。推荐优先测试：

| 类别 | 候选模型 | 用途 |
|---|---|---|
| GPT 系列 | GPT-4.1 Mini / GPT-4o Mini | 商用强基线，成本可控 |
| Gemini 系列 | Gemini 2.5 Flash / Flash Lite | 视觉理解强、调用成本低 |
| Qwen VL 系列 | Qwen3-VL-235B / Qwen3-VL-32B / Qwen2.5-VL-72B | 国产视觉大模型主力候选 |
| GLM V 系列 | GLM-5V Turbo / GLM-4.6V / GLM-4.5V | 国产视觉模型替代路线 |
| Kimi / Seed / MiniMax | Kimi K2.6、ByteDance Seed、MiniMax-01 等 | 低成本扩展对比 |

DeepSeek 当前在 OpenRouter 上主要是文本模型，适合做 text-only reasoning / policy sanity check，不适合作为多图视觉主评测模型。

## 6. 建议申请的 OpenRouter API 额度

当前已有评估集约为 **380 条结构化视觉样本**。如果横评 6-8 个模型，约需要 **2280-3040 次视觉模型调用**；本轮不跑 Claude，主要选择 GPT/Gemini mini/flash 和国产低成本 VLM，因此预算可以比上一版明显降低。

| 档位 | 预计调用量 | 覆盖范围 | 建议额度 |
|---|---:|---|---:|
| 最小 smoke | 约 400 次 | 5 个模型 × 80 条样本，快速筛掉明显不合适的模型 | **$30-$50** |
| 标准横评（推荐） | 约 3000 次 | 6-8 个模型 × 当前完整评估集，形成公司汇报和论文 baseline 表 | **$100-$150** |
| 扩展横评 | 5000+ 次 | top 模型复测、扩展 QA set、不同 prompt 版本和成本-效果曲线 | **$200-$300** |

建议本轮先申请 **$150-$200**。如果预算审批偏保守，最低也应先给 **$50** 做 smoke test；如果希望同时覆盖更多国产模型和 top-2 复测，建议按 **$300** 准备。

## 8. 当前结论

PIWM 已经完成第一轮可验证实验闭环：

- 数据能生成；
- 模型能训练；
- 训练后能力有明显提升；
- 未来反应验证路线已经跑通；
- 下一步需要做模型横向对比，确定最佳基座。

因此，继续投入 OpenRouter 的目的很明确：

```text
不是验证 PIWM 能不能做，而是找出 PIWM 应该基于哪个 VLM 做得最好、最便宜、最适合产品化。
```
