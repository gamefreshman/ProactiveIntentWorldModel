# 米塔世界数字人导购终端宣发展示拍摄方案

本文用于重新界定“摆拍”的目的：它不是复刻 2026-04-29 旧版拍摄清单里的状态机，也不是当前论文训练数据的唯一来源，而是围绕公司“米塔世界”销售终端中的智能 3D 数字人导购，拍一组能对外说明产品效果的应用场景素材。

## 1. 当前问题

2026-04-29 版拍摄清单可以作为早期现场执行参考，但它有两个问题：

1. 状态机过时：其中的状态命名和动作设计不是当前 PIWM compact visual-state / action-realization schema。
2. 展示对象错位：如果用真人销售完整扮演导购，画面中心会变成“真人导购会不会卖货”，而不是“米塔世界数字人终端如何辅助门店销售”。

因此，本次宣发拍摄不能继续按“真人销售 A/B 对照短片”理解，而应改成：

```text
顾客在真实商品前表现出可见状态
-> 米塔世界数字人终端识别并给出导购建议/话术
-> 顾客或现场人员按建议完成一次低压力互动
-> 展示顾客反应和终端能力
```

## 2. 两条拍摄线必须分开

| 拍摄线 | 主要用途 | 画面主角 | 能否写进论文数据 | 说明 |
|---|---|---|---|---|
| PIWM 研究数据线 | 训练 / 评估 / QA-reviewed evidence | 顾客状态 + 导购动作后果 | 可以，但必须标注清楚 | 关注画面证据、状态标签、动作后果，不强调品牌露出 |
| 米塔世界宣发展示线 | 公司产品演示 / 对外宣传 / 投融资展示 | 数字人终端 + 顾客互动 | 不直接写成训练数据 | 关注终端能力、产品体验、数字人导购价值 |

关键原则：宣发线素材可以作为 demo 和应用场景展示，但不能直接写成“模型训练集已经人工验证通过”。如果后续要入库为论文真实数据，必须再走 PIWM 标注和 QA 流程。

## 3. 宣发展示的核心叙事

不要拍成“真人导购上前推销”。推荐叙事是：

```text
门店中有顾客正在看商品。
米塔世界数字人终端从画面或导购视角理解顾客状态。
终端给出一句具体可执行的导购话术和动作建议。
顾客得到更自然、更低压力的引导。
```

对外可以表达为：

> 米塔世界智能 3D 数字人导购终端，可以在店内视觉场景中理解顾客当前状态，并给出具体、克制、可执行的导购建议，帮助门店提升服务响应质量。

不要表达为：

- “数字人已经完全替代真人销售”
- “所有场景都已真实部署验证”
- “摆拍数据就是论文 QA-pass 数据”
- “模型直接预测顾客未来视频”

## 4. 推荐画面结构

每条宣发 clip 建议 12-20 秒，比训练数据 10 秒略长，方便观众看懂终端界面。

| 时间 | 画面 | 目的 |
|---|---|---|
| 0-4 秒 | 顾客在商品区自然浏览、犹豫、比较或试用 | 让观众看到真实销售终端场景 |
| 4-7 秒 | 镜头切到或带到米塔世界数字人终端屏幕 | 建立“数字人终端在场” |
| 7-11 秒 | 屏幕显示数字人识别到的顾客状态和建议话术 | 展示 AI 能力，不只展示角色形象 |
| 11-16 秒 | 顾客或现场人员按建议进行低压力互动 | 展示建议可执行 |
| 16-20 秒 | 顾客继续停留、回应、上手或完成询问 | 展示效果，但不要夸张成交 |

如果不方便现场实时接入系统，可以拍成“模拟 UI + 后期合成”。但画面必须明确：这是米塔世界数字人终端的产品演示，不是普通真人导购摆拍。

## 5. 数字人终端应如何出现在画面里

推荐三种镜头，不推荐只拍真人销售。

### 镜头 A：终端在店内真实位置

- 画面里同时出现商品区、顾客、终端屏幕。
- 数字人形象在屏幕中，不需要真人导购站到画面中心。
- 顾客可以看向屏幕，或现场工作人员根据屏幕提示辅助。

适合展示：

```text
数字人导购终端不是聊天机器人，而是嵌入销售现场的服务入口。
```

### 镜头 B：屏幕特写 + 顾客状态小窗

屏幕 UI 可以展示：

- 顾客状态：例如“价格犹豫 / 正在比较 / 正在试用”
- 观察证据：例如“反复看价签、停留较久、身体仍朝向商品”
- 建议动作：例如“先做价值对比，不要强推”
- 可说话术：例如“我可以帮您把这两款的价格和材质差别简单对比一下。”

注意：UI 上不要堆内部 enum，比如 `high_hesitation`、`A2_offer_value_comparison`。对外展示要用自然中文。

### 镜头 C：数字人输出后的人机协同

两种拍法都可以：

1. 数字人直接对顾客说话。
2. 数字人给店员/现场人员建议，由人低压力执行。

如果公司认为真人销售会稀释数字人价值，优先使用第 1 种；如果现场不适合直接让屏幕对顾客说话，可以使用第 2 种，但真人只做执行者，不做主角。

## 6. 推荐 4 个宣发场景

这 4 个场景与当前 PIWM 标签体系兼容，但对外表达不使用内部状态机名。

### 场景 1：价格犹豫，数字人建议价值对比

顾客表现：

- 在两个商品之间停留。
- 反复看价签或活动牌。
- 身体仍朝向商品，没有离开。

终端识别：

```text
顾客可能对价格和价值差异存在犹豫。
```

数字人建议：

```text
不要直接强推。先帮助顾客比较价格、材质、功能和适用场景。
```

数字人话术：

```text
“我看您在比较这几款。如果愿意，我可以帮您把价格、材质和适用场景简单对比一下，您不用马上决定。”
```

预期效果：

- 顾客继续停留。
- 视线回到商品。
- 愿意听一个简短比较。

### 场景 2：试用评估，数字人建议短演示

顾客表现：

- 正在上手试用、翻看或测试商品。
- 视线集中在商品细节。
- 手部动作明显。

终端识别：

```text
顾客已经有兴趣，正在确认使用体验。
```

数字人建议：

```text
只演示一个最关键细节，不做长篇推销。
```

数字人话术：

```text
“我只演示一个最关键的地方，您看这个细节是不是符合您的使用习惯。”
```

预期效果：

- 顾客继续试用。
- 关注点从犹豫转向体验确认。
- 不产生被强推感。

### 场景 3：回避或准备离开，数字人建议不打扰

顾客表现：

- 不看导购或屏幕。
- 身体转离商品。
- 手从商品附近收回。

终端识别：

```text
顾客当前互动意愿较低。
```

数字人建议：

```text
不要追问，不要强推。只保留低压力可用入口。
```

数字人话术：

```text
“您先慢慢看，需要的时候我再帮您。”
```

预期效果：

- 顾客不会被进一步打扰。
- 可以继续浏览或自然离开。
- 展示数字人不是只会推销，也会判断“不该介入”。

### 场景 4：接近柜台，数字人建议开放式提问

顾客表现：

- 主动走向柜台或服务区。
- 视线朝向服务点。
- 可能手持商品或准备询问。

终端识别：

```text
顾客接近决策或咨询节点。
```

数字人建议：

```text
用开放式问题确认顾客最关心的一个点。
```

数字人话术：

```text
“您现在主要想先确认哪一点？我可以按一个点帮您看，不会耽误太久。”
```

预期效果：

- 顾客给出一个需求点。
- 互动进入明确服务流程。

## 7. 拍摄执行建议

### 人员

- 顾客演员：主角，负责表现可见状态。
- 数字人终端：画面核心产品。
- 真人店员：可选，只能作为执行辅助，不要变成主角。

### 机位

推荐主机位：

```text
顾客 + 商品 + 终端屏幕 同时可见的中景
```

补充机位：

- 顾客手部 / 商品细节
- 终端 UI 特写
- 顾客看向屏幕或继续看商品的反应

### UI 展示

屏幕 UI 至少展示三层：

```text
观察到什么 -> 判断是什么 -> 建议怎么说/怎么做
```

例如：

```text
观察：顾客反复查看价格，停留较久
判断：可能在比较价格和价值
建议：先做价值对比，避免强推
话术：我可以帮您把这几款简单对比一下
```

## 8. 与 PIWM 数据字段的对应关系

宣发 UI 可以映射到 PIWM 字段，但不要直接暴露内部标签。

| 对外 UI | PIWM 内部字段 | 对外表达 |
|---|---|---|
| 观察到什么 | `visual_state` | 顾客持续停留、看价签、试用、回避等 |
| 顾客状态 | `aida_stage` + `latent_state` | 正在比较、正在犹豫、准备咨询 |
| 建议策略 | `best_action` | 价值对比、开放式提问、短演示、保持观察 |
| 具体话术 | `best_action_realization.utterance` | 可直接说出口的一句话 |
| 具体动作 | `best_action_realization.physical_action` | 站位、距离、手势、是否后退 |
| 结果展示 | `visible_reaction` | 顾客继续停留、回应、后撤、离开 |

## 9. 交付物

建议交付两类素材。

### 宣发素材

- 4 个核心场景，每个 1 条 20 秒以内成片。
- 每个场景 1 条竖版短视频剪辑。
- 每个场景 3-5 张关键帧截图。
- 每个场景 1 张终端 UI 特写图。

### 可入库素材

如果这些素材后续要进入 PIWM 数据集，还需要额外交付：

- 原始无字幕视频。
- 终端 UI 合成前的干净画面。
- 每条 clip 的 `visual_state`、`best_action_realization` 和 `visible_reaction` 标注。
- QA 报告，确认画面是否真的支持标签。

## 10. 最终判断

这次拍摄应该从“真人销售摆拍”改为“数字人终端应用演示”。真人可以存在，但不能承担产品主角。画面必须让观众看懂：

```text
米塔世界数字人终端看到了什么
它判断顾客处于什么服务状态
它建议怎么说、怎么做
顾客因此获得了更自然的服务体验
```
