## Random-candidate Baseline Estimate

Current measured random-candidate macro F1 is `0.2811`.

Under the stage-conditioned candidate sizes above, rough random accuracy is:

```text
(13 rows with 4 candidates * 1/4 + 17 rows with 3 candidates * 1/3) / 30
= 0.2972
```

This is an accuracy expectation, not an exact macro-F1. Exact macro-F1 should be recomputed after implementation because class-wise prediction distribution changes with the candidate list.

## always-X Baseline

The always-X baselines become less semantically clean when candidates are stage-conditioned:

- `always-Greet` can only be valid for `attention` and `action`; on `interest/desire` it is outside the candidate set.
- `always-Elicit` is invalid for `desire/action`.
- `always-Inform` is invalid for `action`.
- `always-Recommend` is invalid for `attention`.
- `always-Hold` remains valid for all stages.

If eval requires predictions to be in the candidate set, invalid always-X predictions should count as automatic misses. That makes always-X useful mainly as a sanity check, not as a strong baseline. A better baseline after implementation is `random-valid-candidate` plus `stage-majority-valid-act`.

## 5-act Invariant Test

The invariant test should not change conceptually:

```text
Operational files must contain zero Reassure in candidate specs / prompts / targets.
```

The 5-act set remains unchanged. Only the per-stage candidate subset changes.
